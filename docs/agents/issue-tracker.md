# Issue Tracker

Issues tracked as local markdown files under `.scratch/<feature>/`.

When creating a new issue:
1. Create a folder for feature/workstream if needed.
2. Add a markdown issue file with title, problem, acceptance criteria.
3. Add triage state/category labels in frontmatter.

Issue template:
- Start from `.scratch/_templates/issue.md` to keep assumptions, alternatives, verification, and next-step guidance consistent.
